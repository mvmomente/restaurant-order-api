# MVP - Sistema de Gestão de Restaurante

## Qual problema o sistema resolve?

O sistema resolve a dificuldade de controlar pedidos dentro de um restaurante, centralizando o gerenciamento de produtos, pedidos e usuários em uma única plataforma.

Com ele será possível:

* Registrar pedidos.
* Acompanhar o status dos pedidos.
* Organizar o fluxo entre garçom, cozinha e gerente.
* Gerenciar produtos do cardápio.
* Controlar acesso através de diferentes perfis de usuário.

---

## Quem usa o sistema?

### Cliente

* Visualizar cardápio.
* Criar pedidos.
* Personalizar itens do pedido.

### Garçom

* Criar pedidos.
* Visualizar pedidos.
* Atualizar status dos pedidos.

### Cozinheiro

* Visualizar pedidos em preparo.
* Acompanhar itens que precisam ser produzidos.

### Gerente

* Gerenciar produtos.
* Visualizar pedidos.
* Acompanhar operação do restaurante.

### Administrador

* Gerenciar usuários.
* Possuir acesso total ao sistema.

---

## O que o sistema NÃO faz?

Nesta primeira versão (MVP), o sistema não terá:

* Integração com meios de pagamento.
* Controle de estoque.
* Emissão de notas fiscais.
* Relatórios avançados.
* Dashboard com métricas.
* Integração com aplicativos de entrega.
* Notificações por e-mail ou SMS.
* Sistema de reservas de mesas.
* Controle financeiro.
* Impressão automática de pedidos.
* Aplicativo mobile.

---

## Escopo da Primeira Entrega

O MVP será considerado concluído quando for possível:

1. Cadastrar usuários.
2. Realizar login.
3. Cadastrar produtos.
4. Listar produtos.
5. Criar pedidos.
6. Adicionar produtos ao pedido.
7. Alterar status dos pedidos.
8. Visualizar pedidos.
9. Controlar permissões por perfil de usuário.
